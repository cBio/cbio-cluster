MSKCC cBio Cluster User Guide

Overview

The cBio cluster consists of 30 Exxact dual-socket E5-2665 2.4GHz nodes (32 hyperthreads/node) with 256GB of memory. Each node is configured with 4x NVIDIA GTX-680 [20 nodes] or 4x GTX-Titan [10 nodes] GPUs.

System configuration

There are 30 compute nodes in the cluster, each containing

  • 2x Intel Xeon E5-2665 2.4GHz CPUs in hyperthreaded mode, providing a total of 32 threads per node
  • 256GB memory
  • 4x NVIDIA GPUs per node, either 4GB GTX-680s [20 nodes] or 6GB GTX-Titans
  • 10GE ethernet interfaces The compute nodes are provided by Exxact Corp

Storage subsystem

Overview

Fast local filesystem storage for the cluster is provided by a GPFS filesystem hosted on Dell servers. This filesystem is intended for caching local datasets and production storage, rather than long-term data archival. This storage is shared by all cBio groups, and quotas will be enforced after a friendly user period ends.

Batch queue system overview

The batch queue system is still undergoing trials and tuning. We currently use the slurm batch queue system (see below).

Submitting jobs to the batch queue system

Submitting GPU jobs to the batch queue system

Example:

#!/bin/tcsh
#  Batch script for MPI GPU job on the cbio cluster
#  utilizing 4 GPUs, with one thread/GPU
#
#SBATCH --job-name=jobname
#SBATCH --time=12:00:00
#SBATCH --partition=gpu
#SBATCH --nodes=1
#SBATCH --ntasks-per-node=4
#SBATCH --output=jobname-%j.stdout
#SBATCH --error=jobname-%j.stderr
#SBATCH --gres=gpu:4

Interactive jobs

You can use srun --pty -I --partition=active to start interactive jobs.

Example: To start an interactive job with time limit of 1 hour, using up to 1GB of memory, requesting one GPU (GTX-Titan), and starting the tcsh shell: 

srun --pty -I --mem=1000M --partition=active --gres=gpu:1 --constraint=gtxtitan tcsh
